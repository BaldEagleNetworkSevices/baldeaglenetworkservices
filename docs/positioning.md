# Positioning Lock: Recovery-First IT with Security Enforcement

## Core Positioning

- Recovery-first IT with security enforcement.
- Primary value statement: If everything fails, your systems can be recovered.
- Offer sequence: assess recovery risk, prove restore capability, enforce controls that support recovery.

## Primary Conversion Path

- Primary conversion path is always the Recovery Assessment.
- Primary conversion CTA label should remain focused on Recovery Assessment across header, key sections, and final CTAs.
- Any supporting proof content must route users back to Recovery Assessment.

## Operator-Led Delivery Model

- Work is operator-led and accountability is direct.
- Avoid language that implies call-center routing, anonymous dispatching, or outsourced ownership.
- Messaging should reinforce that the same operator is responsible for assessment findings and follow-through.

## Proof Positioning

- The backup recovery case study is a proof asset, not a side resource.
- Proof language should stay specific: backups are not reliable unless tested and restorable.
- Proof content should support conversion into Recovery Assessment, not replace it.
